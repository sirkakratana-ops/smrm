\# 🌾 Srea Man Agriculture Invoice Bot (ស្រែមាន)

A secure, lightweight, and cost-effective Telegram Bot designed for **\*\*ហាងកសិកម្ម ស្រែមាន\*\*** that allows B2B agricultural customers to instantly look up their purchase history and outstanding debt profiles using their phone numbers. 

By leveraging Telegram's native UI and secure protocol, this system completely bypasses the traditional Apple App Store and Google Play Store ecosystem—saving deployment time, avoiding developer fees, and removing friction for non-technical users.

\#\# 🗄️ 1\. Database Schema & Setup

Log into your \*\*Supabase Dashboard\*\*, open the \*\*SQL Editor\*\*, paste the following script into the console, and click \*\*Run\*\*. This establishes your relational tables with native PostgreSQL internationalization support for Khmer script Unicode characters (\`TEXT\`).

\`\`\`sql  
\-- 1\. Create Customers Table  
CREATE TABLE customers (  
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,  
    name TEXT NOT NULL,  
    phone TEXT UNIQUE NOT NULL  
);

\-- Create a database index for instant lookup queries  
CREATE INDEX idx\_customers\_phone ON customers(phone);

\-- 2\. Create Invoices Master Table  
CREATE TABLE invoices (  
    id TEXT PRIMARY KEY, \-- Stores manual/digital invoice number (e.g. '20260312')  
    customer\_id BIGINT REFERENCES customers(id) ON DELETE CASCADE NOT NULL,  
    invoice\_date TIMESTAMP WITH TIME ZONE NOT NULL,  
    total\_amount NUMERIC(12, 2\) NOT NULL,  
    status TEXT DEFAULT 'Paid' \-- e.g., 'Paid', 'Debt' (ជំពាក់)  
);

\-- 3\. Create Invoice Line Items Table  
CREATE TABLE invoice\_items (  
    id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,  
    invoice\_id TEXT REFERENCES invoices(id) ON DELETE CASCADE NOT NULL,  
    product\_name TEXT NOT NULL, \-- e.g., 'Padan (ដប)', 'Galaxy (500g)'  
    quantity INT NOT NULL,  
    unit\_price NUMERIC(12, 2\) NOT NULL  
);

⚠️ **Important Security Step:** Ensure **Row Level Security (RLS)** is either disabled for these tables under your Supabase Database Settings or create a policy that explicitly grants read permissions (SELECT) to public anon API calls so your Node.js application can access the rows.

## **🚀 2\. Server Deployment Configuration (Render)**

This application utilizes a production-ready **Webhook callback bridge** rather than resource-heavy long-polling. Your Render instance will sleep when idle and wake up instantly when a client messages your bot.

1. Go to **Render.com**, click **New \+**, select **Web Service**, and link your GitHub repository.  
2. Configure the deployment parameters exactly as follows:  
   * **Runtime:** Node  
   * **Build Command:** npm install  
   * **Start Command:** npm start  
3. Click on the **Advanced** section and add the following three **Environment Variables**:

| Variable Environment Key | Description / Value Source |
| :---- | :---- |
| TELEGRAM\_BOT\_TOKEN | Secure API token provided by Telegram's @BotFather |
| SUPABASE\_URL | API Endpoint URL (Found in Supabase Project Settings \-\> API) |
| SUPABASE\_KEY | Secured Public anon key (Found in Supabase Project Settings \-\> API) |

## **📲 3\. User Authentication & String Normalization**

To enforce security and privacy, customers are blocked from typing alphanumeric phone strings. The interface enforces verification via Telegram's native secure interactive button payload.

### **The Verification Workflow**

1. The customer interacts with the bot by sending the /start command.  
2. The bot displays a single native button prompt: 📲 ចែករំលែកលេខទូរស័ព្ទ (Share Contact).  
3. Telegram securely transmits the user's registered phone number to the backend.  
4. **Data Sanitization Layer:** Telegram frequently passes numbers differently depending on regional carriers (e.g., dropping prefixes or keeping whitespaces like \+855 15 612 512). The script cleans and structures this text before running queries:

JavaScript  
// Clean text layer inside index.js  
let phone \= ctx.message.contact.phone\_number;  
phone \= phone.replace(/\[^0-9+\]/g, ''); // Removes all whitespaces or special characters

if (\!phone.startsWith('+')) {  
    phone \= '+' \+ phone; // Restores safe global prefix lookup standard (+855...)  
}
